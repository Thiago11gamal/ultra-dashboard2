# Auditoria de bugs — 2026-05-02

Este relatório foi gerado a partir de verificação automática completa com ESLint e build de produção.

## Comandos executados
- `npm run lint`
- `npm run build`

## Resultado geral
- **Build de produção**: passou.
- **Qualidade de código (lint)**: falhou com **122 problemas** (**112 erros** + **10 warnings**).

## Bugs/problemas encontrados (prioridade alta)

### 1) SetState dentro de `useEffect` (risco de render em cascata)
Ocorrências críticas detectadas por `react-hooks/set-state-in-effect`:
- `src/components/Header.jsx` (linha ~68)
- `src/components/MonteCarloGauge.jsx` (linha ~57)
- `src/hooks/useMonteCarloStats.js` (linha ~265)
- `src/hooks/useSubscription.js` (linha ~11)

**Impacto**: rerenders desnecessários, possibilidade de loop e inconsistência de estado em cenários reais.

### 2) Componentes criados durante render
Ocorrências críticas detectadas por `react-hooks/static-components`:
- `src/components/charts/EvolutionChart/MonteCarloEvolutionChart.jsx` (`CustomTooltip` criado no render e usado em `<Tooltip content={<CustomTooltip />} />`)
- `src/components/charts/EvolutionChart/WeeklyEvolutionView.jsx` (mesmo padrão, 2 ocorrências)

**Impacto**: perda de memoização, reset de estado interno de componentes e regressão de performance.

### 3) Erros de imutabilidade em hooks
Ocorrência crítica detectada por `react-hooks/immutability`:
- `src/components/PomodoroTimer.jsx` (atribuição em `syncChannelRef.current` reportada como mutação indevida no contexto da regra)

**Impacto**: comportamento inesperado em execução e incompatibilidade com otimizações do React Compiler.

### 4) Preservação de memoização quebrada
Ocorrência crítica detectada por `react-hooks/preserve-manual-memoization`:
- `src/hooks/useMonteCarloStats.js` (`useMemo` com dependências divergentes da inferência do compilador)

**Impacto**: memoização manual inconsistente, risco de valores stale ou recomputações erradas.

### 5) Grande volume de variáveis/imports não usados
Erros massivos de `no-unused-vars` espalhados por múltiplos módulos (`components`, `hooks`, `engine`, `store`, `utils`).

**Impacto**: dívida técnica alta, maior risco de bugs latentes e manutenção difícil.

## Sinais de risco de performance no build
Apesar do build passar, o bundler reportou chunk grande:
- `dist/assets/charts-*.js` acima de 1MB (minificado)

**Impacto**: piora de tempo de carregamento inicial e interatividade em dispositivos mais lentos.

## Recomendações objetivas (ordem sugerida)
1. Corrigir primeiro todos os erros de `react-hooks/*` (são os mais propensos a bug funcional).
2. Extrair `CustomTooltip` para fora do corpo dos componentes (ou usar render function estável com memo).
3. Revisar `useMemo`/`useEffect` no `useMonteCarloStats` para alinhar dependências reais.
4. Fazer limpeza sistemática de `no-unused-vars` por domínio (engine -> hooks -> UI).
5. Aplicar code splitting adicional nas rotas/charts para reduzir bundle principal.

## Observação final
A base **compila**, mas **não está saudável** para manutenção/escala devido ao volume de erros de lint e regras de hooks quebradas.
